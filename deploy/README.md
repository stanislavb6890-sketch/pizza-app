# Развертывание на VPS

## Требования

- VPS с Ubuntu 20.04 или 22.04
- Минимум 2 GB RAM, 20 GB диска
- Root доступ
- Домен с A-записью, указывающей на IP сервера

## Быстрый старт

### Способ 1: Загрузка на сервер

```bash
# На сервере
wget https://raw.githubusercontent.com/your-repo/pizza-delivery/main/deploy/setup-vps.sh
chmod +x setup-vps.sh
sudo ./setup-vps.sh
```

### Способ 2: Копирование

```bash
# Локально
scp deploy/setup-vps.sh root@your-server-ip:/root/

# На сервере
ssh root@your-server-ip
chmod +x setup-vps.sh
./setup-vps.sh
```

## Что устанавливается

| Компонент | Версия | Порт |
|-----------|--------|------|
| Node.js | 20.x | 3000 (app) |
| MySQL | 8.x | 3306 |
| Redis | latest | 6379 |
| Nginx | latest | 80, 443 |
| PM2 | latest | - |
| Certbot | latest | - |

## Переменные окружения

После установки файл `.env.production` содержит:

```bash
NEXT_PUBLIC_APP_URL=https://your-domain.com
NEXTAUTH_URL=https://your-domain.com
NEXTAUTH_SECRET=<auto-generated>
JWT_SECRET=<auto-generated>
DATABASE_URL=mysql://pizza_user:<password>@localhost:3306/pizza_delivery
REDIS_URL=redis://localhost:6379
NODE_ENV=production
```

## Управление

### Приложение (PM2)

```bash
pm2 status                    # Статус
pm2 logs pizza-delivery       # Логи
pm2 restart pizza-delivery    # Перезапуск
pm2 stop pizza-delivery       # Остановка
pm2 monit                     # Мониторинг
```

### Nginx

```bash
sudo systemctl status nginx   # Статус
sudo systemctl reload nginx   # Перезагрузка конфига
sudo nginx -t                 # Проверка конфига
```

### База данных

```bash
sudo systemctl status mysql   # Статус MySQL
sudo mysql -u root            # Вход в MySQL
```

### Redis

```bash
sudo systemctl status redis   # Статус
redis-cli ping                # Проверка (должен вернуть PONG)
```

## Обновление проекта

```bash
cd /var/www/pizza-delivery
git pull origin main
npm ci
npx prisma migrate deploy
npm run build
pm2 restart pizza-delivery
```

## SSL-сертификат

Автообновление настроено через crontab (каждый день в 3:00).

Проверить:
```bash
sudo certbot certificates
sudo certbot renew --dry-run
```

## Файрвол (UFW)

```bash
sudo ufw status               # Статус
sudo ufw allow 8080           # Открыть порт
sudo ufw delete allow 8080    # Закрыть порт
```

## Устранение неполадок

### Приложение не запускается

```bash
pm2 logs pizza-delivery --lines 100
cd /var/www/pizza-delivery && npm run build
```

### Ошибка 502 Bad Gateway

```bash
pm2 status                    # Приложение запущено?
sudo systemctl status nginx   # Nginx работает?
```

### Проблемы с SSL

```bash
sudo certbot certificates
sudo certbot renew --force-renewal
```

### Нет подключения к БД

```bash
sudo systemctl status mysql
mysql -u pizza_user -p pizza_delivery
```

## Донастройка

### Добавление переменных окружения

```bash
cd /var/www/pizza-delivery
nano .env.production
# Добавьте переменные
pm2 restart pizza-delivery
```

### Масштабирование

```bash
pm2 scale pizza-delivery 2    # 2 экземпляра
```

### Настройка бекапов БД

```bash
# Установка
apt install automysqlbackup

# Ручной бекап
mysqldump -u pizza_user -p pizza_delivery > backup.sql

# Восстановление
mysql -u pizza_user -p pizza_delivery < backup.sql
```
